Explore ES6

ES6 represents the biggest change to the language since ES3 was released in 1999. You can take advantage of ES6 features today using transpilers, like Babel. If you are using something like Browserify in your JavaScript build pipeline, adding Babel transpilation takes only a couple of minutes. There is support for pretty much every common Node.js build system like Gulp, Grunt, WebPack and many others. ES6 official specification can be found here.

Variables

In ES5, var x declares variable x in execution context. Usually it is declared in global scope. If the declaration appears in a function - a local variable is declared. Within the body of a function, a local variable takes precedence over a global variable with the same name.

In ES6, we have let and const, which are block-scoped naturally - their scopes are the innermost enclosing blocks. The difference between these two: const variable can't be reassigned.

Arrow Functions

Arrows are a function shorthand using the => syntax. They are syntactically similar CoffeeScript. Unlike functions, they behave differently in a number of important ways:

  • Lexical this binding – The value of this inside of the function is determined by where the arrow function is defined not where it is used.
  • Can’t change this – The value of this inside of the function can’t be changed, it remains the same value throughout the entire lifecycle of the function.

  • Arrow functions are designed to be lightweight and cannot be used as generators. Using the yield keyword in ES6 will throw an error. Use ES6 generators instead.

Arrow functions are especially handy for short callbacks that only return results of expressions.

Default, Rest and Spread

The default function parameters allow formal parameters to be initialized with default values if no value or undefined is passed. The rest parameters allow your functions to have variable number of arguments without using the arguments object. The rest parameter is an instance of Array so all the array methods just work. The spread operator can be used as a butter knife alternative over using .apply. There is no need for a context either. You just append three dots ... to the array, just like with the rest parameter. The spread operator can turn the contents of its operand into Array elements.

forEach, for-in and for-of

forEach is an method that is available only in Array objects. It allows you to iterate through elements of an array. When invoked it takes a callback function and invokes the callback once for every array element. The callback can access both index and value of the array elements. forEach is available only for looping arrays.

for in is used to loop through properties of an object. It can be any object. It allows you to access the keys of the object but doesn’t provide reference to the values. In JavaScript object properties themselves have internal properties. One of the internal properties is [[Enumerable]]. for in will only walkthrough a property if it has [[Enumerbale]] set to true. It's not used to iterate elements of an collection rather used to iterate properties of objects.

for of is a new way for iterating collections. Earlier you had to use for or while loop to iterate through elements of an collection. For for of to work on an collection, the collection must have an [Symbol.iterator] property.

ES5:
ES6:
Don't use for-in unless you use it with safeguards or are at least aware of why it might bite you. Use for-of for loops (ES6), forEach for Arrays (ES5) or old-fashioned for loop for any situation. for-in could iterate in an arbitrary order. Furthermore, other properties on the array (such as length) could also be included in the iteration (older version of Internet Explorer did this). Finally, the i iteration variable is a String, not a Number. https://strongloop.com/strongblog/introduction-to-es6-iterators/

Modules

In ES6 each module is defined in its own file. The functions or variables defined in a module are not visible outside unless you explicitly export them. This means that you can write code in your module and only export those values which should be accessed by other parts of your app. To export certain variables from a module you just use the keyword export. Similarly, to consume the exported variables in a different module you use import.

ES6:
Tools such as browserify or webpack let you use ES6 syntax to create modules, making the code you write future-proof. http://www.sitepoint.com/understanding-es6-modules/

Maps, WeakMaps and Sets

Developers usually just use regular JavaScript objects when they want maps. Stuff is mapped in an object by using strings as a key and this is possible with any kind of Object in Javascript.

WeakMaps are different from Maps: you can only use objects as keys for a WeakMap - no numbers, no strings or other primitive types as keys. Objects used as keys by a WeakMap can be garbage collected if there are no other references to the object elsewhere in your application. Another important difference is that WeakMaps only have a subset of the methods maps have and cannot be iterated over in a for...of loop. The methods a WeakMap has are limited, to get,set, has, delete and clear.

Set is a unique list of values enumerable in insertion order. It does not provide set operation functionality like finding differences, intersections and unions. It’s simply a unique list.

ES6:
Map entries are always iterated in insertion order. This is in contrast with Object.keys loops which follow an arbitrary order. http://www.2ality.com/2015/01/es6-maps-sets.html

Proxies

ES6 proxies sit between your code and an object. A proxy allows you to perform meta-programming operations such as intercepting a call to inspect or change an object’s property. Proxy can be helpful for libraries which implement data binding. Because a handler can be hooked to the data model, there is no need to use an alternative syntax (e.g. explicit set to change a value) or to continuously track the change (e.g. dirty state) to modify the model.

ES6:
Think at least twice before using proxies - more often than not they are overkill, and there is usually a simpler and more efficient way to achieve the same thing. https://ponyfoo.com/articles/es6-proxies-in-depth

Symbols

Symbols are a new primitive type, which allow properties to be keyed by either string (as in ES5) or symbol. You create symbols via the factory function Symbol(). It also has an optional string-valued parameter that lets you give the newly created Symbol a description. That description is used when the symbol is converted to a string (via toString() or String()).

You can’t coerce(implicitly convert) symbols to strings, it will throw exception! http://blog.keithcirkel.co.uk/metaprogramming-in-es6-symbols/

Promises

Promises finally have arrived natively in JavaScript! Promises give us a way to handle asynchronous processing in a more synchronous fashion. They represent a value that we can handle at some point in the future. And, better than callbacks, Promises give us guarantees about that future value, specifically, that no other registered handlers of that value can change it (the Promise is immutable) and that we are guaranteed to receive the value, regardless of when we register a handler for it, even if it's already resolved (no more race conditions).

ES6:
Having Promises natively, you can avoid using libraries like Q, Bluebird, RSVP and others. Introduction to ES6 Promises – The Four Functions You Need To Avoid Callback Hell
